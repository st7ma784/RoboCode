# Tank Submissions

Submit your tank here to battle against the sample tanks!

## How to Submit

### Step 1: Create Your Tank Folder

Create a folder with your name:
```
Submissions/
  YourName/
    my_tank.py
    README.md (optional - describe your tank)
```

### Step 2: Add Your Tank

Put your tank Python file in your folder. Make sure it:
- Has a class with your tank logic
- Follows the RoboCode tank structure
- Has a unique name

### Step 3: Create a Pull Request

1. Fork this repository
2. Add your tank folder
3. Commit with message: "Add [YourName]'s tank"
4. Create a Pull Request

## What Happens Next?

When you create a PR, our automated system will:

1. **Run Battles** - Your tank fights against sample tanks
2. **Generate Video** - Create a video of the battles
3. **Create Scoresheet** - Show your wins, losses, and stats
4. **Post Results** - Comment on your PR with results

## Battle Format

Your tank will fight:
- SittingDuck (warmup)
- SpinBot (easy)
- WallsBot (medium)
- TrackerBot (hard)
- ChampionBot (expert)

Each battle runs for 100 ticks or until one tank is destroyed.

## Scoring

Points are awarded for:
- **Win**: 10 points
- **Damage Dealt**: 1 point per 10 damage
- **Survival**: 5 points if you survive
- **Accuracy**: Bonus points for hit percentage

## Rules

1. **No Cheating** - Don't modify game engine or access opponent code
2. **Be Original** - Write your own tank (don't copy samples)
3. **Be Respectful** - Keep code and comments appropriate
4. **Have Fun** - This is about learning!

## Tips for Success

1. **Test Locally First** - Make sure your tank works
2. **Start Simple** - Get a basic tank working before adding complexity
3. **Learn from Defeats** - Study the videos to improve
4. **Iterate** - Submit improved versions!
